How It Works

One FRAME. Run it anywhere. It behaves the same everywhere.

A FRAME is a single, self-contained instance you control, and you can run it in whatever way fits your setup:

  • Open it directly in a web browser
  • Run it as a native desktop application
  • Launch it from the command line
  • Package it as an executable
  • Deploy it in a container
  • Boot it directly on bare metal

Regardless of where it runs, it behaves the same way, because every FRAME uses the same kernel and protocols.

Apps Connect, They Don't Own

Applications connect to your FRAME instead of owning your data.

Applications (dApps) don't own your data.

Instead, they connect to your FRAME, request capabilities, and operate on the same shared foundation.

Portable by Design

Copy it, clone it, sync it, or host it yourself.

A FRAME can be copied, backed up, cloned, synchronized with other trusted FRAME instances, or hosted entirely on your own hardware.

That lets everything from a personal laptop to a distributed, user-owned cloud run the exact same software, without changing how applications are built.
